1. NGÔN NGỮ LẬP TRÌNH

1.1. Định nghĩa Ngôn ngữ Lập trình

Ngôn ngữ lập trình (Programming Language) là một tập con của ngôn ngữ máy tính.

Ngôn ngữ lập trình là một dạng ngôn ngữ được thiết kế và chuẩn hóa (đối lập với ngôn ngữ tự nhiên) để truyền các chỉ thị cho máy tính (hoặc máy khác có bộ xử lí).

1.2. Phân loại Ngôn ngữ Lập trình

1.2.1. Ngôn ngữ lập trình cấp thấp

  • Ngôn ngữ máy (Machine Language): Là ngôn ngữ làm việc của CPU, được biểu diễn dưới dạng số nhị phân. 00101011.
  • Hợp ngữ (Assembly Language): Là dạng ngôn ngữ máy đã cải tiến bằng cách dùng mã gợi nhớ (mnemonic) thay cho số nhị phân để biểu diễn mã tác vụ và toán hạng.

1.2.2. Ngôn ngữ lập trình cấp cao (High-level Language)

  • Ngôn ngữ lập trình cấp cao giúp con người viết chương trình dễ dàng, nhanh chóng hơn hợp ngữ.
  • Nó sử dụng lệnh dễ hiểu, gần gũi với ngôn ngữ của con người hoặc công thức toán.
  • Chương trình viết bằng ngôn ngữ cấp cao cần phải dịch trước ra ngôn ngữ máy rồi mới chạy được.

2. CÁC KHÁI NIỆM CƠ BẢN

2.1. Lập trình máy tính (Programming)

Lập trình máy tính (gọi tắt là lập trình) là nghệ thuật cài đặt một hoặc nhiều thuật toán trừu tượng có liên quan với nhau bằng một ngôn ngữ lập trình để tạo ra một chương trình máy tính.

2.2. Thuật toán (Algorithm)

  • Định nghĩa: Thuật toán là tập hợp (dãy) hữu hạn các chỉ thị (hành động) được định nghĩa rõ ràng nhằm giải quyết một bài toán cụ thể nào đó.
  • Độ phức tạp của thuật toán.

2.3. Các tính chất của thuật toán

Thuật toán bao gồm 5 tính chất quan trọng:

  1. Tính chính xác: Quá trình tính toán hay các thao tác máy tính thực hiện phải chính xác.
  2. Tính rõ ràng: Các câu lệnh minh bạch được sắp xếp theo thứ tự nhất định.
  3. Tính khách quan: Được viết bởi nhiều người trên máy tính nhưng kết quả phải như nhau.
  4. Tính phổ dụng: Có thể áp dụng cho một lớp các bài toán có đầu vào tương tự nhau.
  5. Tính kết thúc: Phải hữu hạn các bước tính toán.

2.4. Ví dụ về Thuật toán: Giải phương trình bậc nhất $ax + b = 0$

Đầu vào: $a, b$ thuộc $\mathbb{R}$.

Đầu ra: nghiệm phương trình $ax + b = 0$.

Các bước logic:

  • Nếu $a = 0$:
    • Nếu $b = 0$ thì PT có nghiệm bất kì (Vô số nghiệm).
    • Nếu $b \neq 0$ thì PT vô nghiệm.
  • Nếu $a \neq 0$:
    • PT có nghiệm duy nhất $x = -b/a$.

3. CÁC BƯỚC XÂY DỰNG CHƯƠNG TRÌNH VÀ BIỂU DIỄN THUẬT TOÁN

3.1. Các bước xây dựng chương trình

Các bước xây dựng một chương trình máy tính bao gồm:

  1. Xác định vấn đề (bài toán).
  2. Lựa chọn phương pháp giải.
  3. Xây dựng thuật toán/thuật giải.
  4. Cài đặt chương trình.
  5. Hiệu chỉnh chương trình (Tìm và sửa Lỗi cú pháp và Lỗi ngữ nghĩa).
  6. Thực hiện chương trình.

3.2. Biểu diễn Thuật toán

Thuật toán có thể được biểu diễn bằng 3 cách chính:

  • Ngôn ngữ tự nhiên.
  • Lưu đồ - Sơ đồ khối.
  • Mã giả.

3.2.1. Sử dụng Ngôn ngữ tự nhiên (Ví dụ: Giải $ax + b = 0$)

Đầu vào: $a, b$ thuộc $\mathbb{R}$.

Đầu ra: nghiệm phương trình $ax + b = 0$.

  1. Nhập 2 số thực $a$ và $b$.
  2. Nếu $a = 0$ thì:
    1. Nếu $b = 0$ thì:
      • PT vô số nghiệm.
      • Kết thúc thuật toán.
    2. Ngược lại (tức $b \neq 0$):
      • PT vô nghiệm.
      • Kết thúc thuật toán.
  3. Ngược lại (tức $a \neq 0$):
    1. PT có nghiệm.
    2. Giá trị của nghiệm là $x = -b/a$.

3.2.2. Sử dụng Lưu đồ - Sơ đồ khối

Các khối cơ bản trong Lưu đồ:

Khối Ý nghĩa
Khối giới hạn Chỉ thị bắt đầu và kết thúc.
Khối vào ra Nhập/Xuất dữ liệu.
Khối lựa chọn Tùy điều kiện sẽ rẽ nhánh.
Khối thao tác Ghi thao tác cần thực hiện (ví dụ: Tính $x = -b/a$).
Đường đi Chỉ hướng thao tác tiếp theo.

Minh họa Lưu đồ (Giải $ax+b=0$):

(Bắt đầu) $\to$ (Nhập a,b) $\to$ (Lựa chọn $a=0?$)
$\to$ (Nếu Đúng) $\to$ (Lựa chọn $b=0?$)
$\quad \to$ (Nếu Đúng) $\to$ (Xuất “VSN”) $\to$ (Kết thúc)
$\quad \to$ (Nếu Sai) $\to$ (Xuất “VN”) $\to$ (Kết thúc)
$\to$ (Nếu Sai, $a \neq 0$) $\to$ (Tính $x = -b/a$) $\to$ (Xuất $x$) $\to$ (Kết thúc)

3.2.3. Sử dụng Mã giả (Pseudocode)

Mã giả vay mượn ngôn ngữ nào đó (ví dụ Pascal) để biểu diễn thuật toán.


Đầu vào: a, b thuộc R.
Đầu ra: nghiệm phương trình ax + b = 0.

If a = 0 Then
Begin
    If b = 0 Then
        Xuất “Phương trình vô số nghiệm”
    Else
        Xuất “Phương trình vô nghiệm”
End
Else
    Xuất “Phương trình có nghiệm x = -b/a”

3.3. Cài đặt thuật toán bằng C và C++ (Ví dụ)

Cài đặt bằng C:


#include <stdio.h>
#include <conio.h>

void main()
{
    int a, b;
    printf("Nhap a, b: ");
    scanf("%d%d", &a, &b);
    
    if (a == 0)
        if (b == 0)
            printf("Phuong trinh VSN");
        else
            printf("Phuong trinh VN");
    else
        printf("x = %.2f", -float(b)/a);
}

Cài đặt bằng C++:


#include <iostream>
using namespace std;

void main()
{
    int a, b;
    cout<<"Nhap a, b: ";
    cin>>a>>b;

    if (a == 0)
        if (b == 0)
            cout<<"Phuong trinh VSN";
        else
            cout<<"Phuong trinh VN";
    else
        cout<<"x = "<<-float(b)/a);
}

4. GIỚI THIỆU TỔNG QUAN VỀ NGÔN NGỮ LẬP TRÌNH C/C++

4.1. Lịch sử Ngôn ngữ C/C++

  • Ngôn ngữ C: Được phát triển bởi Dennis Ritchie tại Bell Telephone năm 1972. Tiền thân của C là ngôn ngữ B (tạo bởi Ken Thompson).
  • C là ngôn ngữ lập trình có cấu trúc và phân biệt chữ Hoa - thường (case sensitive).
  • Ngôn ngữ C++: Là phần mở rộng của C. Phát triển bởi Bjarne Stroustrup (Bell Laboratories) vào đầu thập niên 1980.
  • C++ cung cấp khả năng lập trình hướng đối tượng.
  • C++ được gọi là Ngôn ngữ lai (Hybrid language).

4.2. Đặc điểm và Ưu điểm của C/C++

  • C thường được dùng để lập trình hệ thống.
  • C được dùng như là một ngôn ngữ trung gian vì nó có thể xuất thành dạng tập tin object hay ngôn ngữ máy.
  • Ưu điểm:
    • Rất mạnh và linh động, có khả năng thể hiện bất cứ ý tưởng nào.
    • Được sử dụng rộng rãi bởi các nhà lập trình chuyên nghiệp.
    • Có tính khả chuyển (portable), ít thay đổi trên các hệ thống máy tính khác nhau.
    • Rõ ràng, cô đọng.
    • Lập trình đơn thể (modular), tái sử dụng thông qua hàm.

4.3. Môi trường phát triển tích hợp (IDE)

Môi trường phát triển tích hợp (Integrated Development Environment - IDE) bao gồm các thành phần:

  • Biên tập chương trình nguồn (Trình EDIT).
  • Biên dịch chương trình (Trình COMPILE).
  • Chạy chương trình nguồn (RUNTIME).
  • Sửa lỗi chương trình nguồn (DEBUG).

Quá trình chuyển đổi tập tin: .C/.CPP $\to$ .OBJ $\to$ .EXE.

Một số môi trường lập trình thường dùng: Borland C++ 3.1, Microsoft Visual C++ 6.0, Microsoft Visual Studio 2010/2012 (Win32 Console Application).

5. BỘ TỪ VỰNG VÀ CẤU TRÚC CHƯƠG TRÌNH C++

5.1. Bộ từ vựng của C/C++

5.1.1. Các ký tự được sử dụng

  • Bộ chữ cái 26 ký tự Latinh (A–Z, a–z).
  • Bộ chữ số thập phân (0–9).
  • Các ký hiệu toán học: + – * / = < > ( ).
  • Các ký tự đặc biệt: . , : ; [ ] % \ # $ ‘.
  • Ký tự gạch nối _ và khoảng trắng ‘.

5.1.2. Từ khóa (Keyword)

Là các từ dành riêng trong ngôn ngữ và không thể được sử dụng để đặt tên cho biến, hàm, tên chương trình con.

Thông dụng
const, enum, signed, struct, typedef, unsigned
Kiểu dữ liệu
char, double, float, int, long, short, void
Rẽ nhánh
case, default, else, if, switch
Vòng lặp
do, for, while
Lệnh nhảy
break, continue, goto, return

5.1.3. Tên/Định danh (Identifier)

Là một dãy ký tự dùng để chỉ tên một hằng số, hằng ký tự, tên một biến, một kiểu dữ liệu, một hàm hay thủ tục.

Quy tắc đặt tên:

  • Không được trùng với các từ khóa.
  • Được tạo thành từ các chữ cái và các chữ số, nhưng bắt buộc chữ đầu phải là chữ cái hoặc _.
  • Số ký tự tối đa trong một tên là 255 ký tự.
  • Được dùng ký tự _ chen trong tên nhưng không cho phép chen giữa các khoảng trắng.
  • C++ phân biệt chữ hoa chữ thường (Ví dụ: A khác a).

Ví dụ tên hợp lệ: GiaiPhuongTrinh, Bai_Tap1.

Ví dụ tên không hợp lệ: 1A (bắt đầu bằng số), Giai Phuong Trinh (có khoảng trắng).

5.1.4. Dấu chấm phẩy (;) và Câu chú thích

  • Dấu chấm phẩy (;): Dùng để phân cách các câu lệnh.
  • Câu chú thích (Comment): Đặt giữa /* */ (nhiều dòng) hoặc // (một dòng). Trình biên dịch sẽ bỏ qua.

5.1.5. Hằng ký tự và hằng chuỗi

  • Hằng ký tự: Đặt trong dấu nháy đơn. Ví dụ: 'A', 'a'.
  • Hằng chuỗi: Đặt trong dấu nháy kép. Ví dụ: "Hello World!".

5.2. Cấu trúc chương trình C++

C++ được chia nhỏ thành những đơn vị gọi là hàm.

Cấu trúc cơ bản:


#include “…”; // Khai báo file tiêu đề
using namespace std;

int x; // Khai báo biến
void Nhap(); // Khai báo hàm

void main() // Hàm chính
{
    // Các lệnh và thủ tục
}

Giải thích các thành phần chính:

  • Hàm main(): Phải có ít nhất một hàm main(). Hệ điều hành luôn trao quyền điều khiển cho hàm main() khi chương trình thực thi.
  • Dấu phân cách {...}: Dấu { bắt đầu và } kết thúc một hàm (khối lệnh).
  • Trình biên dịch C++ chỉ nhận diện câu lệnh dựa trên dấu chấm phẩy ;.

5.3. Ví dụ minh họa cấu trúc chương trình C++

Chương trình tính tổng hai số nguyên:


#include <iostream>
using namespace std;

void main()
{
    int x, y, tong;
    cout<<"Nhap hai so nguyen: ";
    cin>>x>>y;
    
    tong = x + y;
    
    cout<<"Tong hai so = ";
    cout<<tong;
}

6. BÀI TẬP VÀ ỨNG DỤNG THUẬT TOÁN

6.1. Bài tập Lý thuyết

  2. Các bước xây dựng chương trình.
  3. Các cách biểu diễn thuật toán? Ưu và khuyết điểm của từng phương pháp? Cho ví dụ minh họa.

6.2. Bài tập Thực hành: Công thức và Thuật toán

Bài 6: Tính Tiền và Thuế GTGT

Đầu vào: Tên sản phẩm, Số lượng, Đơn giá.

Công thức:

  • Tiền = Số lượng * Đơn giá.
  • Thuế giá trị gia tăng (VAT) = $10\%$ Tiền.

Lưu đồ: (Nhập) $\to$ (Tính Tiền, VAT) $\to$ (Xuất Tiền, VAT) $\to$ (Kết thúc).

Bài 7: Tính Điểm Trung Bình (ĐTB)

Đầu vào: Điểm thi (T, L, H) và Hệ số (HsT, HsL, HsH).

Công thức:

$$ \text{ĐTB} = \frac{(\text{T} \times \text{HsT}) + (\text{L} \times \text{HsL}) + (\text{H} \times \text{HsH})}{\text{HsT} + \text{HsL} + \text{HsH}} $$

Lưu đồ: (Nhập) $\to$ (Tính ĐTB) $\to$ (Xuất ĐTB) $\to$ (Kết thúc).

Bài 8: Tính Chu vi và Diện tích Hình tròn

Đầu vào: Bán kính $R$.

Hằng số: $\text{PI} = 3.1415$.

Công thức:

  • Chu vi = $2 \times \text{PI} \times R$.
  • Diện tích = $\text{PI} \times R \times R$.

Lưu đồ: (Nhập $R$) $\to$ (Tính Chu vi, Diện tích) $\to$ (Xuất) $\to$ (Kết thúc).

Bài 9: Tính Số nút xe

Đầu vào: Số xe $N$ (gồm 4 chữ số).

Thuật toán (Các bước tính):

  1. Số thứ 4: $n4 = N \pmod{10}$, $N = N / 10$.
  2. Số thứ 3: $n3 = N \pmod{10}$, $N = N / 10$.
  3. Số thứ 2: $n2 = N \pmod{10}$, $N = N / 10$.
  4. Số thứ 1: $n1 = N$.
  5. Số nút $S = (n1 + n2 + n3 + n4) \pmod{10}$.

Lưu đồ: (Nhập $N$) $\to$ (Tính $n4, n3, n2, n1, S$) $\to$ (Xuất $S$) $\to$ (Kết thúc).

Bài 10: Tính Min và Max của hai số nguyên $a, b$

Đầu vào: Đọc $a, b$.

Lưu đồ: (Bắt đầu) $\to$ (Đọc $a, b$) $\to$ (Lựa chọn $a > b$)
$\to$ (Nếu $Đ$): (Xuất $a$ max, $b$ min) $\to$ (Kết thúc).
$\to$ (Nếu $S$): (Xuất $a$ min, $b$ max) $\to$ (Kết thúc).

6.3. Bài tập Mở rộng

  1. Nhập số nguyên $n$, kiểm tra xem $n$ có phải là số chính phương hay không?
  2. Nhập số nguyên $n$, kiểm tra xem $n$ có phải là số nguyên tố hay không?
  3. Nhập vào 3 hệ số $a, b, c$ của phương trình bậc 2 ($ax^2 + bx + c = 0$), cho biết nghiệm của phương trình trên.
  4. Viết chương trình nhập vào một số nguyên dương $N$. Tính và in ra tổng các số nguyên từ 1 đến $N$.
