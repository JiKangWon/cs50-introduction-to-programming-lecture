I. CÂU LỆNH ĐIỀU KIỆN IF

1. Mục đích và Khái niệm

Các câu lệnh điều kiện cho phép chúng ta thay đổi hướng thực hiện của chương trình. Một câu lệnh điều kiện sẽ trả về giá trị đúng hoặc sai.

Ví dụ thực tế: Để xác định một số là số chẵn hay số lẻ:

  • Nhập vào một số.
  • Chia số đó cho 2 để xác định số dư.
  • Nếu số dư là 0, số đó là “SỐ CHẴN”.
  • Ngược lại, số dư không bằng 0, số đó là “SỐ LẺ”.

2. Cú pháp Lệnh IF

Biểu thức logic trong ngoặc đơn () sẽ cho kết quả sai = 0 hoặc đúng ≠ 0. Khối lệnh có thể là Câu lệnh đơn hoặc Câu lệnh phức (được kẹp giữa { và }).

2.1. Dạng 1 (Thiếu)

Chỉ thực hiện hành động nếu điều kiện đúng.

Cú pháp:


if (biểu thức điều kiện)
    <khối lệnh>;

Ví dụ Code (C++):


// Ví dụ 1: Lệnh đơn
if (a == 0)
    cout << "a bang 0";

// Ví dụ 2: Lệnh phức
if (a == 0)
{
    cout << "a bang 0";
    a = 2912;
}

2.2. Dạng 2 (Đủ)

Thực hiện hành động A nếu điều kiện đúng, và hành động B nếu điều kiện sai.

Cú pháp:


if (biểu thức điều kiện)
    <khối lệnh 1>;
else
    <khối lệnh 2>;

Ví dụ Code (C++):


// Ví dụ 1: Lệnh đơn
if (a == 0)
    cout << "a bang 0";
else
    cout << "a khac 0";

// Ví dụ 2: Lệnh phức
if (a == 0)
{
    cout << "a bang 0";
    a = 2912;
}
else
    cout << "a khac 0";

3. Một số Lưu ý quan trọng khi dùng IF

  • Câu lệnh if và if… else là một câu lệnh đơn.
  • Câu lệnh if có thể lồng vào nhau. Trong trường hợp lồng nhau, từ khóa else sẽ tương ứng với if gần nó nhất. 
    
// Ví dụ lồng nhau:
if (a != 0)
    if (b > 0)
        cout << "a != 0 va b > 0";
    else
        cout << "a != 0 va b <= 0"; // else này của if (b > 0)
  • Nên dùng else để loại trừ trường hợp. Điều này giúp chương trình rõ ràng hơn và tránh kiểm tra lại các điều kiện đã bị loại trừ. 
    
// Ví dụ tối ưu hóa giải phương trình bậc hai:
if (delta < 0)
    cout << "PT vo nghiem";
else // delta >= 0
    if (delta == 0)
        cout << "PT co nghiem kep";
    else // delta > 0
        cout << "PT co 2 nghiem";
  • Tuyệt đối Không được thêm dấu chấm phẩy (;) sau điều kiện của if. 
    
// VÍ DỤ SAI LOGIC:
if (a != 0); // Lỗi logic: Dấu ; kết thúc lệnh if ngay tại đây
cout << "a khac 0."; // Lệnh này luôn được thực thi

II. CÂU LỆNH RẼ NHÁNH SWITCH

1. Khái niệm và Mục đích

Lệnh switch là một bộ lựa chọn đa hướng. Nó so sánh giá trị của một biểu thức với một danh sách các hằng số nguyên hoặc hằng ký tự.

Khi gặp một sự so sánh khớp, các lệnh kết hợp với hằng đó được thực thi.

Yêu cầu: Biến/Biểu thức trong switch() phải là biến/biểu thức cho giá trị rời rạc (ví dụ: int, char).

2. Cú pháp Lệnh SWITCH

2.1. Dạng 1 (Thiếu)

Cú pháp:


switch (<Biến/BT>)
{
    case <GT1>:
        <Lệnh 1>;
        break;
    case <GT2>:
        <Lệnh 2>;
        break;
    // ...
}

Lưu ý: Lệnh break; được dùng để thoát khỏi khối switch.

Ví dụ Code (C++):


switch (a)
{
    case 1:
        cout << "Mot";
        break;
    case 2:
        cout << "Hai";
        break;
    case 3:
        cout << "Ba";
        break;
}

2.2. Dạng 2 (Đủ - Có default)

Khi giá trị của Biến/BT không khớp với bất kỳ case nào, khối lệnh trong default sẽ được thực thi.

Cú pháp:


switch (<Biến/BT>)
{
    case <GT1>:
        <Lệnh 1>;
        break;
    case <GT2>:
        <Lệnh 2>;
        break;
    // ...
    default:
        <Lệnh n>;
}

Ví dụ Code (C++):


switch (a)
{
    case 1:
        cout << "Mot";
        break;
    case 2:
        cout << "Hai";
        break;
    case 3:
        cout << "Ba";
        break;
    default:
        cout << "Ko biet";
}

3. Một số Lưu ý quan trọng khi dùng SWITCH

  • Câu lệnh switch là một câu lệnh đơn và có thể lồng nhau.
  • Các giá trị trong mỗi trường hợp (case) phải khác nhau.
  • Nguyên tắc thực thi: switch sẽ nhảy đến case tương ứng và thực hiện đến khi nào gặp break hoặc cuối switch sẽ kết thúc.
  • Tận dụng tính chất khi bỏ break;: Có thể gộp nhiều case vào chung một khối lệnh thực thi. 
    
// Ví dụ: Gộp số lẻ/số chẵn:
switch (a)
{
    case 1:
    case 3:
        cout << "So le";
        break;
    case 2:
    case 4:
        cout << "So chan";
        break;
}

III. KINH NGHIỆM LẬP TRÌNH

1. So sánh IF và SWITCH

Nên chọn cấu trúc phù hợp với loại điều kiện kiểm tra.

Tiêu chí Câu lệnh if Câu lệnh switch
Phạm vi Kiểm tra bất kỳ biểu thức logic nào (so sánh bằng, lớn hơn, nhỏ hơn, điều kiện phức tạp...). Chỉ kiểm tra sự bằng nhau của một biểu thức/biến duy nhất với các giá trị hằng số rời rạc (số nguyên, ký tự).
Tính ứng dụng Phù hợp khi cần kiểm tra các điều kiện không liên tục, so sánh phức tạp (ví dụ: a == 3.14 hoặc a < 10). Ưu tiên sử dụng khi cần kiểm tra nhiều điều kiện so sánh bằng nhau trên cùng một biến.

IV. CÁC THUẬT TOÁN VÀ VÍ DỤ MINH HỌA

Dưới đây là các thuật toán và công thức được cung cấp trong tài liệu cho các bài tập điển hình:

Bài tập 2: Đổi chữ thường sang chữ hoa và ngược lại

Thuật toán: Dựa trên bảng mã ASCII, khoảng cách giữa chữ hoa và chữ thường là 32.


char ch;
// ... (Nhập ch)
if (ch >= 'a' && ch <= 'z') // Nếu là chữ thường
    ch = ch - 32; // Chuyển sang hoa (Trừ 32)
else
    if (ch >= 'A' && ch <= 'Z') // Nếu là chữ hoa
        ch = ch + 32; // Chuyển sang thường (Cộng 32)
        
cout << "Ky tu sau khi doi: " << ch;

Bài tập 3: Giải phương trình bậc nhất $ax + b = 0$

Thuật toán: Cấu trúc if lồng nhau để xử lý các trường hợp của $a$ và $b$.

  • Kiểm tra hệ số $a$:
    • Nếu $a = 0$:
      • Kiểm tra $b$: Nếu $b = 0$ $\to$ Phương trình Vô Số Nghiệm (VSN).
      • Ngược lại ($b \neq 0$) $\to$ Phương trình Vô Nghiệm (VN).
    • Ngược lại ($a \neq 0$):
      • Nghiệm là: $x = \frac{-b}{a}$.
      • Code: cout << "Nghiem = " << float(-b)/a;

Bài tập 5: Tìm số nhỏ nhất (min) trong 4 số $a, b, c, d$

Thuật toán: Sử dụng biến tạm min và cập nhật nó qua các phép so sánh độc lập.


// ... (Nhập a, b, c, d)
int min = a; // 1. Khởi tạo: min = a

if (b < min) // 2. So sánh với b
    min = b;
    
if (c < min) // 3. So sánh với c
    min = c;
    
if (d < min) // 4. So sánh với d
    min = d;

cout << "So nho nhat la: " << min; // 5. Kết quả

Bài tập 7: Tính tiền đi taxi theo số km nhập vào ($n$)

Thuật toán (Bắt buộc phải nhớ công thức):

A. Khai báo hằng số giá tiền:
  • Giá 1km đầu: G1 = 15000 (đ).
  • Giá km thứ 2 đến thứ 5: G2 = 13500 (đ).
  • Giá km thứ 6 trở đi: G3 = 11000 (đ).
B. Công thức tính tổng tiền ($T$) theo số km ($n$):
Điều kiện $n$ Công thức tính $T$ Chi tiết thuật toán
$n = 1$ T = G1 1 km đầu giá $G1$
$2 \leq n \leq 5$ T = G1 + (n – 1) * G2; $G1$ cho km đầu, $(n-1)$ km còn lại giá $G2$
$n > 5$ T = G1 + 4 * G2 + (n – 5) * G3; $G1$ (km 1) + $4 \times G2$ (km 2-5) + $(n-5)$ km còn lại giá $G3$
C. Giảm giá:
  • Nếu $n > 120$ (trên 120km), tổng tiền $T$ được giảm 10%.
  • Công thức giảm giá: T = T * 0.9;.

Tóm tắt và Liên hệ

Cấu trúc điều kiện (if) và cấu trúc rẽ nhánh (switch) giống như các ngã rẽ trong một bản đồ lập trình. Lệnh if cung cấp sự linh hoạt để kiểm tra mọi loại điều kiện phức tạp (giống như chọn đường dựa trên nhiều yếu tố: thời tiết, kẹt xe, khoảng cách). Ngược lại, lệnh switch hiệu quả hơn khi bạn chỉ cần so sánh một điểm dữ liệu với một danh sách cố định các địa điểm đã biết (giống như chọn ga tàu dựa trên số tuyến cụ thể). Việc chọn đúng cấu trúc giúp tối ưu hóa luồng thực thi của chương trình.
