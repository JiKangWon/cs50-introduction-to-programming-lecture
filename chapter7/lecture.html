I. ĐẶT VẤN ĐỀ VÀ KHÁI NIỆM

1. Đặt vấn đề

Xem xét các ví dụ sau trong lập trình:

  • Chương trình cần lưu trữ 3 số nguyên $\rightarrow$ Khai báo 3 biến int a1, a2, a3;.
  • Chương trình cần lưu trữ 100 số nguyên $\rightarrow$ Khai báo 100 biến kiểu số nguyên!
  • Người dùng muốn nhập $n$ số nguyên? $\rightarrow$ Không thực hiện được!

Giải pháp: Cần có một kiểu dữ liệu mới cho phép lưu trữ một dãy các số nguyên và dễ dàng truy xuất.

2. Dữ liệu kiểu Mảng (Khái niệm)

  • Mảng là một kiểu dữ liệu có cấu trúc do người lập trình định nghĩa.
  • Mảng biểu diễn một dãy các biến có cùng kiểu (Ví dụ: dãy các số nguyên, dãy các ký tự,...).
  • Kích thước của mảng được xác định ngay khi khai báo và không bao giờ thay đổi.
  • Ngôn ngữ lập trình C luôn chỉ định một khối nhớ liên tục cho một biến kiểu mảng.

II. KHAI BÁO BIẾN MẢNG

1. Khai báo biến mảng tường minh

Cú pháp khai báo biến mảng một chiều:

$$\text{ }[ \text{} ];$$

Cú pháp khai báo biến mảng nhiều chiều:

$$\text{ }[ \text{}][ \text{}]…[ \text{} ];$$

Ví dụ khai báo:


int Mang1Chieu[10];
int Mang2Chieu[10][20];

Lưu ý quan trọng khi khai báo:

  • Phải xác định <số phần tử> cụ thể (là hằng số) khi khai báo.
  • Tổng số phần tử (mảng nhiều chiều) $= N_1 \times N_2 \times … \times N_n$.
  • Bộ nhớ sử dụng $= \text{} \times \text{sizeof}(\text{})$.
  • Bộ nhớ sử dụng phải ít hơn 64KB (65536 Bytes) (Giới hạn của một số trình biên dịch cũ).
  • Các phần tử trong mảng được đánh chỉ số liên tục từ 0 đến <số phần tử> - 1.

2. Khai báo biến mảng không tường minh (Thông qua typedef)

Cú pháp (mảng một chiều):


typedef <kiểu cơ sở> <tên kiểu mảng>[];
<tên kiểu mảng> <tên biến mảng>;

Ví dụ:


typedef int Mang1Chieu[10];
typedef int Mang2Chieu[10][20];

Mang1Chieu m1, m2, m3; // m1, m2, m3 là các mảng int 10 phần tử
Mang2Chieu m4, m5;     // m4, m5 là các mảng int 10x20

3. Quy tắc xác định số phần tử

Phải xác định cụ thể số phần tử ngay lúc khai báo, không được sử dụng biến.

Cách khai báo SAI (trong C chuẩn cũ):


// Sai: Dùng biến thường
int n1 = 10; 
int a[n1]; 

// Sai: Dùng biến hằng (một số trình biên dịch cũ không hỗ trợ)
const int n2 = 20; 
int b[n2];

Nên sử dụng chỉ thị tiền xử lý #define:


#define n1 10
#define n2 20

int a[n1];      // Tương đương: int a[10];
int b[n1][n2];  // Tương đương: int b[10][20];

4. Khởi tạo giá trị cho mảng

Có các cách khởi tạo sau (Giả sử int a[4];):

  • Khởi tạo giá trị cho mọi phần tử: 
    int a[4] = {2912, 1706, 1506, 1904};
  • Khởi tạo một số phần tử đầu (còn lại gán bằng 0): 
    int a[4] = {2912, 1706}; // a[0]=2912, a[1]=1706, a[2]=0, a[3]=0
  • Khởi tạo giá trị 0 cho mọi phần tử: 
    int a[4] = {0}; // a[0]=0, a[1]=0, a[2]=0, a[3]=0
  • Tự động xác định số lượng phần tử: 
    int a[] = {2912, 1706, 1506, 1904}; // Mảng có 4 phần tử

III. TRUY XUẤT VÀ GÁN DỮ LIỆU KIỂU MẢNG

1. Truy xuất đến một phần tử

Truy xuất được thực hiện thông qua chỉ số của phần tử.

Cú pháp:

$$\text{}[ \text{}][ \text{}]…[ \text{}]$$

Ví dụ: Cho mảng int a[4];

  • Truy xuất hợp lệ: a[0], a[1], a[2], a[3].
  • Chỉ số mảng không hợp lệ (Lỗi thường gặp): a[-1], a[4], a[5],….
    (Ví dụ: a[-1] = 1; hoặc a[4] = 0; sẽ dẫn đến lỗi và phá vỡ bộ nhớ chương trình).

2. Gán dữ liệu kiểu mảng

Không được sử dụng phép gán thông thường giữa hai biến mảng.

Phải gán trực tiếp giữa các phần tử tương ứng (thường dùng vòng lặp).

Ví dụ minh họa việc gán mảng:


#define MAX 3
typedef int MangSo[MAX];
MangSo a = {1, 2, 3}, b;

// b = a; // LỖI: Không thể gán trực tiếp 2 mảng

// Cách gán ĐÚNG:
for (int i = 0; i < MAX; i++) 
{
    b[i] = a[i]; // Gán từng phần tử
}

3. Các lỗi thường gặp khác

  • Khai báo không chỉ rõ số lượng phần tử: 
    // Lỗi: Mảng chưa biết kích thước
int a[]; 
// Sửa: int a[10]; (Phải có kích thước)
  • Khởi tạo cách biệt với khai báo: 
    
// Lỗi:
int a[4]; 
a = {2912, 1706, 1506, 1904}; 

// Sửa: (Phải khởi tạo ngay khi khai báo)
int a[4] = {2912, 1706, 1506, 1904};

IV. TRUYỀN MẢNG CHO HÀM

Khi truyền mảng cho hàm, tham số kiểu mảng trong khai báo hàm giống như khai báo biến mảng.

  • Bản chất: Tham số kiểu mảng truyền cho hàm chính là địa chỉ của phần tử đầu tiên của mảng (truyền tham chiếu/địa chỉ).
  • Khả năng thay đổi: Mảng CÓ THỂ BỊ THAY ĐỔI nội dung sau khi thực hiện hàm.
  • Cách khai báo: Có thể bỏ số lượng phần tử (ở chiều đầu tiên) hoặc sử dụng con trỏ.

Các dạng khai báo hàm tương đương (với int a[10]):


void SapXepTang(int a[10]);
void SapXepTang(int a[]);
void SapXepTang(int *a);

Lưu ý: Số lượng phần tử thực sự của mảng ($n$) thường được truyền qua một biến khác.

Khai báo hàm hoàn chỉnh thường gặp:


#define MAX 100
// 3 cách khai báo tham số mảng a là tương đương
void XuatMang_Cach1(int a[MAX], int n);
void XuatMang_Cach2(int a[], int n);
void XuatMang_Cach3(int *a, int n);

Ví dụ lời gọi hàm:


#define MAX 100

// Hàm NhapMang dùng tham chiếu n để thay đổi số lượng phần tử
void NhapMang(int a[], int &n); 
void XuatMang(int a[], int n);

void main()
{
    int a[MAX], n; // Khai báo mảng 'a' và số phần tử 'n'
    
    NhapMang(a, n); // Truyền mảng 'a' và tham chiếu 'n'
    XuatMang(a, n); // Truyền mảng 'a' và giá trị 'n'
}

V. MỘT SỐ BÀI TOÁN CƠ BẢN TRÊN MẢNG MỘT CHIỀU

Quy ước chung:

  • Số lượng phần tử tối đa: #define MAX 100.
  • Hàm hỗ trợ void HoanVi(int &x, int &y): hoán vị giá trị của hai số nguyên.
  • Hàm hỗ trợ int LaSNT(int n): kiểm tra số nguyên tố (Trả về 1 nếu là SNT, 0 nếu không).

1. Hàm Hỗ trợ

A. Thủ tục HoanVi


void HoanVi(int &x, int &y)
{
    int tam = x;
    x = y;
    y = tam;
}

B. Hàm LaSNT (Kiểm tra Số Nguyên Tố)


int LaSNT(int n)
{
    if (n < 2) return 0; // Số < 2 không phải SNT
    
    int dem = 0;
    for (int i = 1; i <= n; i++)
        if (n % i == 0)
            dem++;
            
    if (dem == 2) // Chỉ chia hết cho 1 và chính nó
        return 1;
    else
        return 0;
}

2. Thao tác Nhập/Xuất Mảng

A. Hàm Nhập Mảng

Ý tưởng: Cho trước mảng có kích thước MAX. Nhập số lượng phần tử thực sự $n$. Dùng vòng lặp for nhập từng phần tử a[i].


#include <iostream>
using namespace std;

void NhapMang(int a[], int &n) // n là tham chiếu
{
    cout << "Nhap so luong phan tu n (n <= " << MAX << "): ";
    cin >> n;
    
    // Nên kiểm tra n hợp lệ
    if (n > MAX || n <= 0) n = 0; 
    
    for (int i = 0; i < n; i++)
    {
        cout << "Nhap phan tu thu " << i << ": ";
        cin >> a[i];
    }
}

B. Hàm Xuất Mảng

Ý tưởng: Dùng vòng lặp for xuất giá trị từng phần tử a[i] từ 0 đến $n-1$.


void XuatMang(int a[], int n)
{
    cout << "Noi dung cua mang la: ";
    for (int i = 0; i < n; i++)
        cout << a[i] << " ";
    cout << "\n";
}

3. Bài toán Tìm Kiếm (Search)

Yêu cầu: Tìm $x$ trong mảng $a$. Nếu có, trả về vị trí đầu tiên. Nếu không, trả về -1.

Hàm Tìm Kiếm (Dùng while):


int TimKiem(int a[], int n, int x)
{
    int vt = 0;
    while (vt < n && a[vt] != x)
        vt++;
        
    if (vt < n) // Tìm thấy (dừng vì a[vt] == x)
        return vt;
    else // Không tìm thấy (dừng vì vt == n)
        return -1;
}

Hàm Tìm Kiếm (Dùng for):


int TimKiem(int a[], int n, int x)
{
    for (int vt = 0; vt < n; vt++)
        if (a[vt] == x)
            return vt; // Tìm thấy, trả về ngay
            
    return -1; // Duyệt hết mảng mà không tìm thấy
}

4. Bài toán Kiểm tra Tính Chất của Mảng

Yêu cầu: Mảng $a$ có phải là mảng toàn các số nguyên tố hay không?

Ý tưởng 1: Đếm số lượng SNT


// Nếu số lượng SNT bằng đúng n thì mảng toàn SNT.
int KiemTra_C1(int a[], int n)
{
    int dem = 0;
    for (int i = 0; i < n; i++)
        if (LaSNT(a[i])) // == 1
            dem++;
            
    if (dem == n)
        return 1;
    return 0;
}

Ý tưởng 2: Đếm số lượng KHÔNG phải SNT


// Nếu số lượng không phải SNT bằng 0 thì mảng toàn SNT.
int KiemTra_C2(int a[], int n)
{
    int dem = 0;
    for (int i = 0; i < n; i++)
        if (!LaSNT(a[i])) // == 0
            dem++;
            
    if (dem == 0)
        return 1;
    return 0;
}

Ý tưởng 3: Tìm phần tử KHÔNG phải SNT (Tối ưu)


// Nếu tìm thấy 1 phần tử không phải SNT $\to$ kết luận ngay.
int KiemTra_C3(int a[], int n)
{
    for (int i = 0; i < n ; i++)
        if (!LaSNT(a[i]))
            return 0; // Tìm thấy 1 số không phải SNT
            
    return 1; // Duyệt hết mảng mà không tìm thấy
}

5. Bài toán Tách và Gộp Mảng

A. Tách các phần tử thỏa điều kiện (Tách SNT)


void TachSNT(int a[], int na, int b[], int &nb)
{
    nb = 0; // Khởi tạo mảng b rỗng
    for (int i = 0; i < na; i++)
        if (LaSNT(a[i]))
        {
            b[nb] = a[i];
            nb++; // Tăng kích thước mảng b
        }
}

B. Tách mảng thành 2 mảng con (SNT và Số còn lại)


void TachSNT2(int a[], int na,
            int b[], int &nb, int c[], int &nc)
{
    nb = 0; // Khởi tạo mảng b (SNT)
    nc = 0; // Khởi tạo mảng c (còn lại)
    
    for (int i = 0; i < na; i++)
        if (LaSNT(a[i]))
        {
            b[nb] = a[i]; 
            nb++;
        }
        else
        {
            c[nc] = a[i]; 
            nc++;
        }
}

C. Gộp 2 mảng thành một mảng


void GopMang(int a[], int na, int b[], int nb,
           int c[], int &nc)
{
    nc = 0;
    // 1. Chuyển mảng a vào c
    for (int i = 0; i < na; i++)
    {
        c[nc] = a[i]; 
        nc++;
    }
    // 2. Chuyển mảng b vào c (tiếp theo)
    for (int i = 0; i < nb; i++)
    {
        c[nc] = b[i]; 
        nc++;
    }
}

6. Bài toán Tìm Giá Trị Lớn Nhất

Ý tưởng: Giả sử max = a[0]. Lần lượt so sánh max với các phần tử còn lại (a[1] đến a[n-1]) để cập nhật max.


int TimMax(int a[], int n)
{
    if (n <= 0) return -1; // Trường hợp mảng rỗng
    
    int max = a[0]; // Giả sử max là phần tử đầu tiên
    for (int i = 1; i < n; i++)
        if (a[i] > max)
            max = a[i]; // Cập nhật max mới
            
    return max;
}

7. Bài toán Sắp Xếp Mảng Tăng Dần (Sorting)

Ý tưởng (Selection Sort): Dùng 2 vòng lặp i và j. Với mỗi $i$, tìm phần tử nhỏ nhất từ $i+1$ đến $n-1$ (do $j$ chạy) và hoán vị nếu $a[i] > a[j]$.


void SapXepTang(int a[], int n)
{
    int i, j;
    for (i = 0; i < n - 1; i++)
    {
        for (j = i + 1; j < n; j++)
        {
            if (a[i] > a[j]) // Cặp nghịch thế (sai thứ tự)
                HoanVi(a[i], a[j]);
        }
    }
}

8. Bài toán Thêm/Xóa Phần tử

A. Thêm một phần tử vào mảng

Ý tưởng (Insertion): "Đẩy" các phần tử từ vị trí $vt$ sang phải 1 vị trí (chạy từ $n$ lùi về $vt+1$). Đưa $x$ vào $a[vt]$. Tăng $n$.


void Them(int a[], int &n, int vt, int x)
{
    if (vt >= 0 && vt <= n && n < MAX) // Kiểm tra vt và mảng chưa đầy
    {
        for (int i = n; i > vt; i--)
            a[i] = a[i - 1]; // Đẩy sang phải
            
        a[vt] = x; // Chèn x
        n++; // Tăng số lượng
    }
}

B. Xóa một phần tử trong mảng

Ý tưởng (Deletion): "Kéo" các phần tử bên phải $vt$ sang trái 1 vị trí (chạy từ $vt$ đến $n-2$). Giảm $n$.


void Xoa(int a[], int &n, int vt)
{
    if (vt >= 0 && vt < n) // Kiểm tra vt hợp lệ
    {
        for (int i = vt; i < n - 1; i++)
            a[i] = a[i + 1]; // Kéo sang trái
            
        n--; // Giảm số lượng
    }
}

VI. BÀI TẬP CỦNG CỐ

Các thao tác cần thực hiện để củng cố kiến thức:

1. Các thao tác nhập xuất

  • a. Nhập mảng.
  • b. Xuất mảng.

2. Các thao tác kiểm tra

  • a. Mảng có phải là mảng toàn số chẵn.
  • b. Mảng có phải là mảng toàn số nguyên tố.
  • c. Mảng có phải là mảng tăng dần.

3. Các thao tác tính toán

  • a. Có bao nhiêu số chia hết cho 4 nhưng không chia hết cho 5.
  • b. Tổng các số nguyên tố có trong mảng.

4. Các thao tác tìm kiếm

  • a. Vị trí cuối cùng của phần tử $x$ trong mảng.
  • b. Vị trí số nguyên tố đầu tiên trong mảng nếu có.
  • c. Tìm số nhỏ nhất trong mảng.
  • d. Tìm số dương nhỏ nhất trong mảng.

5. Các thao tác xử lý

  • a. Tách các số nguyên tố có trong mảng $a$ đưa vào mảng $b$.
  • b. Tách mảng $a$ thành 2 mảng $b$ (chứa các số nguyên dương) và $c$ (chứa các số còn lại).
  • c. Sắp xếp mảng giảm dần.
  • d. Sắp xếp mảng sao cho các số dương đứng đầu mảng giảm dần, kế đến là các số âm tăng dần, cuối cùng là các số 0.

6. Các thao tác thêm/xóa/sửa

  • a. Sửa các số nguyên tố có trong mảng thành số 0.
  • b. Chèn số 0 đằng sau các số nguyên tố trong mảng.
  • c. Xóa tất cả số nguyên tố có trong mảng.
